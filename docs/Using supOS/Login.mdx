## Login

  <div>
    <p>After authorization, you can log in to the supOS platform.</p>
    <ul id="login__ul_vxz_jnw_xtb">
      <li>
        60 days before the expiration, a notification appears on the login page
        to remind you.
      </li>
      <li>
        Within 30 days after the expiration date, the platform notifies you that
        the number of online users is reduced by 10%.
      </li>
      <li>
        30 days after the expiration date, the platform notifies you that only
        admin can access.
      </li>
    </ul>

### Function Menu


  <div>
    <section classname="section" id="function_menu__section_ir5_t4x_xtb">
      <h4 classname="sectiontitle">Collector Node Management</h4>
      <ul id="function_menu__ul_zwh_54x_xtb">
        <li>
          <strong>Authentication</strong>: The supOS platform
          generates a legal UUID upon the access of the data of the collector
          and third-party software. The UUID connects the data and the platform,
          and ensures the security of the connection.
        </li>
        <li>
          <strong>Status</strong>: Manage the status of the
          connection between the collector and the platform, and the source
          point channel of the collector.
        </li>
      </ul>
    </section>
    <section classname="section" id="function_menu__section_m1s_hqx_xtb">
      <h4 classname="sectiontitle">Object Model</h4>
      <p>
        Build models for all workshops, equipment, materials and products in the
        company for easy management.
      </p>
      <ul id="function_menu__ul_lrk_jqx_xtb">
        <li>
          <strong>Object Template</strong>: A collection of
          models that describe information for objects of the same type. Object
          instances of the same type can be created quickly by inheriting object
          templates. An object template can only inherit one other template, and
          an object instance can only inherit one object template. Object
          templates have basic description information, attributes, services,
          events, and subscription functions.
        </li>
        <li>
          <strong>Object Instance</strong>: Represents the
          data information model of the subject with the business relationship
          in the life cycle. Object instances can be quickly created by
          inheriting object templates. Object instances have basic description
          information, attributes, services, events, and subscription functions.
        </li>
        <li>
          <strong>Function Set</strong>: Identify different
          business subsets of object templates and object instances, for batch
          creation and perspective view of function sets. The function set has
          functions such as adding, deleting, and copying by reference.
          Inheritance is unavailable.
        </li>
        <li>
          <strong>Tag Management</strong>: Label and
          categorize all objects in the platform.
        </li>
        <li>
          <strong>Data Set</strong>: User-defined complex data
          structures. Each field represents a specific variable, and each row
          corresponds to a certain object dataset.
        </li>
        <li>
          <strong>Data Group</strong>: Divide object instance
          attributes and alarm data into different groups, and assign these data
          groups to different users or roles for data permission control.
        </li>
        <li>
          <strong>Task Management</strong>: View information
          of subscription tasks and manage them.
        </li>
      </ul>

### System Settings

<div>
    <p>
      Click <img id="system_settings__image_mzm_hjx_xtb" src="https://wordpressfreezonex.oss-cn-hangzhou.aliyuncs.com/platform/6.png" /> and then select <strong>System Information Management</strong>.
    </p>
    <figure classname="fignone" id="system_settings__fig_oc3_vcy_xtb">
      <img id="system_settings__image_hn5_xcy_xtb" src="https://wordpressfreezonex.oss-cn-hangzhou.aliyuncs.com/platform/12.png" />
    </figure>
<ul>
<li><strong>Default Home Page</strong>: Click to select a tab and use its
    background page as the homepage of the supOS platform. Select <strong>Show Navigation</strong>, and the function list is
    displayed on the left side of the homepage.

:::info

You can customize the homepage when you select the default homepage. For details, see <em>SupOS Operation Monitoring Software_User's Manual</em>.

:::
</li>
<li><strong>System Title</strong>: Set the title of the supOS platform.

:::info

<ul id="system_settings__ul_cpf_rgy_xtb">
<li>
The title is displayed on the browser tab during platform operation and management.
        </li>
        <li>
<em>APP title-platform title</em> is displayed during business design.
        </li>
        <li>
<em>To Do_task name(process name)-platform title</em> is displayed when working on a task.
        </li>
      </ul>

:::
 </li>

<li><strong>LOGO</strong>: Set the logo of the supOS platform. The recommended and minimum image
    size is 110 Ã— 36.</li>
<li><strong>System Desktop Configuration</strong>: Select an image to be the
    background of the supOS platform. The configuration is effective for all
    users.</li>
<li><strong>Login Page Configuration</strong>: Set the copyright, logo, background, slogan
    and login title (image) of the login page. See <a href="login.html#system_settings__fig_bhz_thy_xtb">Figure 4</a>.</li>
<li><strong>Storage policy configuration of time series data</strong>: Set the storage duration of the
    time series data history. When the duration exceeds the set value, the
    platform deletes the earliest data and keep the latest.

:::info


<ul id="system_settings__ul_j12_33y_xtb">
<li>The default duration is one day. 0 means permanent storage.</li>
<li>7 GB of space is needed to store 10,000 points of data that changes by the second.
        </li>
</ul>


:::
</li>

<li><strong>Cascading Setup</strong>: Set the cascade information. For details, see <a href="operation_process.html#authorizing_cascading_data_collector__title_p22_ctw_ztb">
      Authorizing Cascading Data Collector</a>.</li> 
<li><strong>Global Function Configuration</strong>: Set the global function that automatically
    runs when the supOS backend service is started.
    <ul id="system_settings__ul_hqx_kjy_xtb">
      <li>
        <strong>window.SUPOS.registerSuposFn</strong> function: Obtains the methods of the global function definition.
      </li>
      <li>
        Once the supOS backend service starts, the <strong>onReady</strong> method of the global function
        will be automatically called, and then you can define specific
        functions. For example:
        <code>
var f1 = function() <br />
var onReady = function() <br />
console.log('This method will be automatically called once supOS starts') <br />
return f1, <br />
onReady
</code>
   
      </li>
    </ul>
    </li>
<li><strong>Business Designer</strong>: Display or hide the <strong>Workflow Management</strong> menu in the business designer. 
We recommend to hide it for servers with low-level configurations.</li>
 <li><strong>App Settings</strong>: Set whether to display the <strong>DataLink Tooltip</strong>
    along with the collected data when running the APP.</li>
 <li><strong>Configure Alarm Colors</strong>: Set the alarm color based on the alarm level. The supOS platform supports up
    to 10 levels with 1 as the lowest.</li>
    </ul>
