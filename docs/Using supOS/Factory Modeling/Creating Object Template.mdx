## Overview

  
  <p>
    Object templates integrate basic and common features of a type of object
    instances. A sub object template and an object instance inherit all
    information from their parent template. When the parent template changes,
    the object instance and the sub template changes too.
  </p>
  

  <ul>
    <li>
      You can create hierarchical object templates by analyzing the commonality
      of similar equipment and staff, thereby improving modeling efficiency.
    </li>
    <li>
      Object templates can derive one or more additional features through
      function sets. Therefore, modeling can be maintained easily and quickly.
    </li>
  </ul>

:::info

<ul className="note-list">
    <li>
      You can search for, copy, edit, import, export and delete custom object
      templates. Fuzzy match is available.
    </li>
    <li>Embedded templates can only be copied.</li>
    <li>
      Inherited templates can only be deleted after you cancel inheritance, and
      the deletion cannot be undone.
    </li>
  </ul>
:::

### Embedded Template


  <ul>
    <li>
      <strong>Entity Template</strong>: Object templates that match with the
      entities in the factory, such as workers, workshops, equipment, products
      and materials.
      <ul className="li-ul">
    <li>
      <strong>Collector Template</strong>: Automatically inherited by collectors
      connected to the supOS platform.
    </li>
    <li>
      <strong>Link Template</strong>: Automatically inherited by the third party
      data connected to the supOS platform.
    </li>
    <li>
      <strong>Video Template</strong>: Automatically inherited by real-time
      video sources connected to collectors.
    </li>
    <li>
      <strong>History Video Template</strong>: Automatically inherited by
      history video sources connected to collectors.
    </li>
    <li>
      <strong>AE Collector Template</strong>:{" "}
    </li>
  </ul>
    </li>
  </ul>
 

  
 

  <ul>
    <li>
      <strong>Form Template</strong>: You can manually create a form structure
      of relational data under the form template. Each property is a column
      header, and each object instance is a row of data.
    </li>
    <li>
      <strong>Relation</strong> <strong>Template</strong>: Builds the relations
      between object models.
       <p className="info">
    For example, associate the attribute <strong>company name</strong> under the
    company template with the attribute <strong>staff name</strong> under the
    staff template to build an employment relation template. Subsequently, you
    can filter the corresponding staff by company name.
  </p>
    </li>
  </ul>


:::info

**Foreign Relation Template** is generated by default.

:::
